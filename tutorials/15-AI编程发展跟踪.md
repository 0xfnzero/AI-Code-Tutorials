# 第十五课：AI 编程发展跟踪与持续迭代手册

> 学习目标：建立一套可重复的更新流程，让本教程能持续跟进 Claude Code、OpenAI Codex、MCP、Skills、Agent SDK 和 AI 编程工作流的发展。
>
> 当前基线：2026-06-20。Claude Code npm latest 为 `2.1.183`，官方变更日志日期为 2026-06-19；OpenAI Codex manual 已复核；MCP 官方规范最新稳定版本为 `2025-11-25`。

## 1. 为什么需要持续跟踪

AI 编程工具的变化速度已经超过传统教程的更新节奏。过去一篇教程可以稳定使用半年，现在命令、配置、模型、权限、安全策略、MCP 协议和 agent 工作流都可能在几周内变化。

本仓库后续更新应遵循一个原则：**教程讲方法，版本细节留校验入口**。也就是说：

- 长期有效的内容：工作流、思维方式、需求拆解、代码审查、测试、安全边界。
- 需要定期复核的内容：CLI 参数、模型名称、配置文件字段、MCP 服务器包名、部署平台步骤、价格和配额。
- 必须引用来源的内容：官方命令、协议规范、安全建议、模型能力和版本发布时间。

## 2. 每月更新检查清单

建议每月进行一次轻量更新，每季度进行一次系统复核。

### 2.1 Claude Code

检查来源：

- [Claude Code Changelog](https://code.claude.com/docs/en/changelog)
- [Claude Code GitHub](https://github.com/anthropics/claude-code)
- npm 包：`npm view @anthropic-ai/claude-code version dist-tags time.modified --json`

检查项：

- 最新版本号和发布时间
- 新增或废弃的 CLI 命令
- 权限、安全、自动模式、Git 操作保护的变化
- subagents、skills、plugins、hooks、IDE 集成和后台任务能力
- Windows、macOS、Linux 的兼容性变化

### 2.2 OpenAI Codex

检查来源：

- [OpenAI Codex manual](https://developers.openai.com/codex/codex-manual.md)
- [Codex overview](https://developers.openai.com/codex)
- Codex CLI：`codex --help`、`codex doctor`、`codex debug models`

检查项：

- Codex CLI、IDE 扩展、Codex app、Codex cloud、GitHub review 的入口变化
- `AGENTS.md`、`config.toml`、rules、hooks、memories 的配置变化
- sandbox mode、approval policy、auto-review、remote environments 的安全变化
- skills、plugins、MCP、subagents、Codex SDK 的能力变化
- ChatGPT 账号、API key、企业工作区和 GitHub/Slack/Linear 集成的权限边界

### 2.3 MCP

检查来源：

- [MCP 官方文档](https://modelcontextprotocol.io/docs/getting-started/intro)
- [MCP 最新规范](https://modelcontextprotocol.io/specification/latest)
- [MCP 规范变更](https://modelcontextprotocol.io/specification/2025-11-25/changelog)

检查项：

- 当前规范版本
- tools、resources、prompts、sampling、elicitation 的变化
- OAuth、OpenID Connect、scope consent 等鉴权变化
- server registry、icons、tool name guidance 等生态能力
- 生产环境安全建议：最小权限、审计日志、工具隔离、用户确认

### 2.4 Skills、Plugins 与可复用工作流

检查来源：

- [Claude Code Skills 文档](https://code.claude.com/docs/en/skills)
- [Agent Skills 说明](https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills)

检查项：

- 自定义命令是否已合并到 Skills 工作流
- `.claude/skills/*/SKILL.md` 的 frontmatter 是否变化
- `.agents/skills/*/SKILL.md` 与 `agents/openai.yaml` 的配置是否变化
- bundled skills 是否新增，例如代码审查、调试、运行验证类技能
- skills 与 subagents、MCP、动态上下文注入的组合方式
- 第三方 skill 的安全审查要求

### 2.5 Agent SDK 与生产级 agent

检查来源：

- [Claude Agent SDK 文档](https://code.claude.com/docs/en/agent-sdk/overview)

检查项：

- TypeScript 和 Python SDK 的安装方式
- Agent loop、permission、hook、checkpoint、observability 的新能力
- 是否适合把教程中的“手动提示词流程”升级为可运行 agent
- 生产环境部署是否需要增加队列、沙箱、日志、成本监控和人工审批

## 3. 2026 年需要重点补齐的教程主题

### 3.1 从 Prompt Engineering 到 Context Engineering

提示词仍然重要，但 AI 编程的关键已经转向“上下文工程”：

- 项目规则：`CLAUDE.md`、`AGENTS.md`、架构说明、命名规范
- 可执行上下文：测试命令、lint 命令、启动命令、数据库迁移命令
- 任务上下文：验收标准、失败样例、截图、日志、复现步骤
- 工具上下文：MCP、浏览器、数据库、CI、监控和设计工具

后续应在第 7、9、12、14 课补充“如何组织上下文”的专门章节。

### 3.2 Skills 与项目级工作流

旧教程中很多“复制粘贴式提示词模板”可以逐步升级为 Skills：

- `/code-review`：固定审查范围、输出格式和风险分级
- `/debug`：固定复现、假设、实验、验证流程
- `/release-check`：固定测试、构建、变更日志、安全检查
- `/docs-update`：固定中英文同步、链接校验、术语统一

建议新增一课专门讲 `.claude/skills/`、`.agents/skills/`、`SKILL.md` 写法和团队共享策略。

### 3.3 Codex 与 Claude Code 双工具路线

本仓库不能只覆盖单一工具。后续内容应明确区分：

- Claude Code 路线：`CLAUDE.md`、Claude Code CLI、MCP、subagents、Claude 生态工作流。
- Codex 路线：`AGENTS.md`、Codex CLI/App/IDE/Cloud、GitHub review、sandbox、approval、skills/plugins。
- 通用路线：上下文工程、代码审查、测试验证、安全边界、MCP、团队工作流。

每个新增章节应标注适用工具：Claude Code、Codex、或通用 AI coding agent。

### 3.4 MCP 生产安全

MCP 不再只是“让 AI 连接工具”的入门概念。生产环境必须补充：

- 每个 MCP server 的权限边界
- 工具调用前后的审计日志
- 高风险工具的人类确认
- 外部内容的 prompt injection 防护
- token、cookie、数据库连接串的最小暴露
- server 名称、tool 名称和参数 schema 的稳定性

第 13 课后续应从“能跑起来”升级到“能安全上线”。

### 3.5 多 agent 协作的验证机制

多 agent 的价值不在于“角色更多”，而在于并行探索和交叉验证。后续第 10 课应强调：

- 一个 agent 负责实现，另一个 agent 负责审查
- 一个 agent 分析需求，另一个 agent 写测试
- 长任务拆成可验收的小任务，而不是一次性放权
- subagent 结果必须回到主流程做整合和验证

## 4. 内容更新流程

每次更新建议按以下步骤执行：

1. 确认本次主题：CLI、MCP、Skills、Agent SDK、项目实战、安全或部署。
2. 读取官方来源，记录具体日期、版本和链接。
3. 搜索仓库中的过期内容，例如 `2025`、旧模型名、旧命令、旧部署平台。
4. 先更新中文，再同步英文；或反向进行，但必须保持目录一致。
5. 对命令类内容至少执行 `--help`、`npm view` 或官方文档核对。
6. 在 `CHANGELOG.md` 记录更新日期、来源和影响范围。
7. 如果只是趋势判断，不写成确定事实；明确标注“建议”“观察”“需要复核”。

## 6. 推荐给读者的长期学习方式

- 每周用一个真实项目练习 AI 辅助开发，而不是只收藏提示词。
- 每月复盘一次：哪些任务适合 agent，哪些任务必须人工设计。
- 保留失败案例：错误修改、误删文件、误判需求、测试遗漏，这些比成功案例更能提升能力。
- 对 AI 输出保持工程化验证：测试、类型检查、代码审查、安全扫描和手动验收都不能省。
